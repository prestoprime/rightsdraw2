Input and Output

Load

The User can load MCO documents on Rightsdraw in OWL/XML serialization. RDF/XML serialization is also accepted only if the suitable plugin for MCO serialization conversion is installed.

Export

The User can get out a copy of MCO documents from Rightsdraw in OWL/XML serialization. RDF/XML serialization is also available as option only if the suitable plugin for MCO serialization conversion is installed.

Derived text

The User has also the possibility to get out a narrative textual version automatically derived from MCO documents.

Import from archive

The User can import MCO documents from a connected preservation system, such as P4, according to the APIs specified for P4, using the identifier of the archival information package as key. This use is for supporting the case in which modifications to rights archived on the content preservation are handled by means of Rightsdraw editing services.

Submit to archive

The User can submit MCO documents to a connected preservation system, such as P4, according to the APIs specified for P4. This is implemented either using the identifier of the archival information package as key, for updating an already existing item, or by wrapping the MCO document within a METS container as submission information package, in the case of new submission.
