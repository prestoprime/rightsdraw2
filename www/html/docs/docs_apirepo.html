In/Out Context interfaces

The In/Out context, with prefix /cgi-bin/rightsdraw2/pprights, provides the following Interfaces:

delete export guiadm import load submit

Name: delete

Description Deletes the given OWL file from the User's repository

Method: GET

Parameters:

  • instance - name of OWL file

Name: export

Description Serves the requested OWL file to User

Method: GET

Parameters:

  • instance - name of OWL file

Name: guiadm

Description Serves to the User, as html output, a number of forms for managing the User's repository (export, delete, load) and for interacting with P4 (submission, import)

Method: GET

Parameters: None

Name: import

Description Given the identifier of a P4 AIP, it imports the rights part of it into User's repository and makes it available for viewing/editing.

Method: GET

Parameters:

  • identifier - ID of a P4 AIP

Name: load

Description Uploads an OWL file to the User's repository

Method: POST

Parameters:

  • instance - name of OWL file

Name: submit

Description Submits the given instance to P4. If an AIP identifier is already known for this instance, the update interface will be used. Otherwise a new SIP is created, containing a Descriptive Metadata section (DMD secion in Mets) filled with the Dublin Core Metadata associated to the main IPEntity as AnnotationProperties, and submitted to the P4 ingest interface for rights-only SIPs. The jobid received from P4 is returned.

Method: GET

Parameters:

  • instance - name of OWL file
