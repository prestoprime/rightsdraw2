Check with and Search

The Check-with and search context provides the means to make rights specific queries and examine their results.

Check-with

With the check-with operation the User defines a query as a target permission and finds out which MCO documents contain Permissions compatible (i.e. same or wider) to the target one. This operation is typical of rights clearance activities.

The comparison between two permissions must return a boolean result:

  • true - when the candidate permission is equal or wider (less restrictive) than the target permission

  • false - when the candidate permission is narrower (more restrictive) than the target permission

For aim of efficiency the check-with is implemented in two steps:

  • rights indexing - performed on the set of candidate permissions, it doesn't need to be repeated until the candidate permissions are modified

  • rights comparison - performed between the target permission and indexed candidate permissions

The check-with operation can be performed on the MCO documents on the user's repository, or remotely on those saved on the connected preservation system, such as P4, provided that the Rightsdraw components for rights indexing and comparison are installed there.

The can create his/her target permission either making use of the defined key patterns rights (KPR) or by means of the more general form.

Search

With the search operation the User looks for MCO documents containing elements, i.e. individuals and their properties, matching the input fields.

Rights information availability in the archive

This operation is conceived for getting information from the connected preservation system, such as P4, about which archival items do have and which ones lack rights information associated. This is to support various archive ingestion scenarios, depending on when (before, during or after) rights information is ready with respect to the ingestion of the audiovisual material.
